NUIPages插件

示例:
代码： 
                    <div class="NUIPages" data-url="./data/Linkage.json" data-total="123" data-options="{success:test,page_size:20,pageSizeField:'pageSize',pageIndexField:'pageIndex',error:test2}" style="text-align: center;"></div>
                    <script type="text/javascript">
                        function test(data,pageIndex){
                            alert(pageIndex);
                        }
                        function test2(e){
                            alert(e);
                        }
                    </script>
说明：

分页插件采用ajax的交互方式，通过绑定申明的方式把count以及url和请求成功后的success函数绑定到dom中，这样方便后台人员直接绑定，而不关注具体的实现逻辑。ajax请求分页数据的时候，在数据根层级有total这个字段返回当前数据的总条数。如：{data:[...],total: "123"}。

NUIPages API列表

属性 参数 说明
NUIPages - 申明该class表明仅使用框架分页的插件
data-url 是一个字符串,这个url即及请求后台分页数据的接口 详细数据可参见demo示例中数据
data-total 是一个数字字符串，是当前分页数据的总条数 详细数据可参见demo示例中数据
data-options

1、page_size(每页多少条,默认每页10条）

2、num_display(翻个个数的设置，默认是10个）

3、pageSizeField(设置传到后台的参数域值，表示每页多少条数据，默认值pageSize)

4、pageIndexField(设置传到后台的参数域值，表示当前是第几页，默认值pageIndex)

5、success函数，即发送ajax请求后，成功时调用的函数。有两个参数，第一个是请求数据data，另一个数据是当前页面pageIndex

6、error函数，即发送ajax请求后，失败时调用的函数，只有一个参数，即e，错误对象

 详细数据可参见demo示例中数据)

备注： 该插件会每次根据回归的total数据，重新刷新这个分页。所以需要返回的数据中，在根级目录必要要有total字段，否则插件渲染会失败。
